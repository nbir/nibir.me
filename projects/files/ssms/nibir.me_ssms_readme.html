README

Smarter Space Management System


Project files

The project files can be found in the SSMS\project folder.

It is available in the deployable ssms.war package and ssmsEAR.ear package, which can be directly deployed as an application on WebSphere Application Server or Apache Tomcat server.

The WebSphere AST workspace is also available in the \workspace folder. Unzip the workspace.zip file to the desired location and choose it as the workspace.

Database configuring

The application database and the contained tables can be created by running the SQL query in the db_and_tables.sql file in the SSMS\database folder.

The application requies to have an administrator user before its first run. Initial administrator user can be created by running the SQL query in the db_initial_data.sql file. The default administrator username is 'administrator, and password is 'password'.

Optional test data can be enteres by running the SQL query in the db_test_data.sql file.

Executing the ssms ddl file creates the database and loads initial data.

Project documents

The Project Report, Presentation, and User Reference can be found in the SSMS\document folder.

System design diagrams

The Component Diagram, Class Diagram, Use Case Diagram, Sequence Diagram, and ER Diagram can be found in the SSMS\design folder.

Test cases

Test Cases for various components can be found in the SSMS\test cases folder.

Changing the database connection

The database connection settings can be modified by changing the dbcon.jspf file in the \include folder of the application. The content of the file looks like this :

con=DriverManager.getConnection("jdbc:db2://localhost:50000/ssms", "db2admin", "password");

'localhost' can be replaced with the desired machine name, containing the database; '50000' with the configured port on that machine; 'db2admin' with the desired username; 'password' with the desired password.

Changing total space available for storage

The total space available for storage can be modified by changing the totalSpace.jspf file in the \include folder of the application. The content of the file looks like this :

totalSpace=10000;

'10000' can be replaced with the desired total space.

Changing host email details

The host email details defines the email settings from which the company user will be notified. This can be modified by changing the hostEmai.jspf file in the \include folder of the application. The content of the file looks like this :

host="smtp.gmail.com";

user="ssms.abc";

pass="smartsms";

The 'host' variable holds the smtp host address, from which the email will be sent. The 'user' variable holds the username, and 'pass' holds the password for the username.

Adding signer to local trust store

Using JavaMail to send mail through a remote SMTP server may cause a SSL Handshake Failure. The error message looks like this:

CWPKI0022E: SSL HANDSHAKE FAILURE: A signer with SubjectDN "CN=smtp.gmail.com, O=Google Inc, L=Mountain View, ST=California, C=US" was sent from target host:port "null:null". The signer may need to be added to local trust store "E:/web/IBM/WebSphere/AppServer/profiles/AppSrv01/config/cells/nibirNode01Cell/nodes/nibirNode01/trust.p12" located in SSL configuration alias "NodeDefaultSSLSettings" loaded from SSL configuration file "security.xml". The extended error message from the SSL handshake exception is: "No trusted certificate found".

To add the signer to the local trust store, open 'Administrative console' and go to 'Security'->'SSL certificate and key management'. Under 'Related Items', click on 'Key stores and certificates'. Click on 'NodeDefaultTrustStore'. Under 'Additional Properties', click on 'Signer certificates'. Click 'Recieve from port'. Fill in the host URL and port number, and choose an alias of your choice. Then click on 'Retrive signer information'. Confirm the information and click 'OK'.
